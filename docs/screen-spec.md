## 0. 확정 모델 요약 (모든 화면의 전제)

- **타깃**: 기관(B2B/유치원) 우선.
- **정보구조**: `모임(Group) > 이벤트(Event) > 앨범(Album)`.
- **인증(제작자)**: 닉네임 + 4자리 PIN.
- **권한**: 등급 없음. 로그인한 모임 멤버는 누구나 업로드·검수·공개 가능.
- **모임 진입점 2개(분리)**
  - **선생님 초대** = 로그인 제작자를 모임에 합류시키는 입구. **모임 단위**(모임 비밀번호 + 참여 링크).
  - **학부모 공개** = 무로그인 뷰어(조회·다운로드만). **모임 단위** 공유 링크 + **학부모 전용 비밀번호**(모임 비밀번호와 별개, 생성 시 자동). 비번 입력 후 **공개된 이벤트 목록**에서 선택.
- **공개(publish) 단위**: **이벤트별** (검수 → 공개 요약 → 공개하기). **학부모 접근(공유) 단위**: **모임별**(공개된 이벤트만 목록 노출).
- **BE/AI 미구현**: FE는 [API 명세서](./api-spec.md) 계약에 목 데이터로 개발.

## 2. 사용자/플로우

### 2-A. 제작자(로그인) 플로우
```
로그인 진입(01)
 ├─ 계정 생성(01-2) ─┐
 └─ 로그인(01-1) ────┴─▶ 홈/내 모임(02)
                              ├─ 모임 만들기(03) ─▶ 모임 상세(05)
                              ├─ 모임 참여(02-1, 모달) ─▶ 모임 상세(05)
                              └─ 모임 선택 ─────────▶ 모임 상세(05)
모임 상세(05, =이벤트 목록)
 ├─ 선생님 초대하기(초대 화면) ── 모임 비밀번호 + 참여 링크 + 카카오톡
 ├─ 이벤트 생성(06-M, 모달) ─▶ 빈 이벤트(06-E)
 └─ 이벤트 선택 ─▶ 이벤트 상세(08) 또는 빈 이벤트(06-E)
빈 이벤트(06-E) ─▶ 사진 업로드(06-U) ─[사진 분류하기]→ 이벤트 상세(08)
이벤트 상세(08, =앨범 그리드)
 ├─ 앨범 선택 ─▶ 앨범 상세(09)
 │                 └─ 옮기기 ─▶ 사진 이동 바텀시트(09-1)
 ├─ + 사진 추가 ─▶ 사진 업로드(06-U)
 └─ 공개 요약(14) ─[공개하기]→ 공개 상태로 전환
설정 ─▶ 프로필 편집(닉네임/PIN)
```

## 3. 화면별 명세

> 표기: **node** = Figma 노드 id, **API** = 소비 엔드포인트(상세는 [API 명세서](./api-spec.md)).

### 01. 로그인 진입 · node `211:1343`
- **목적**: 앱 첫 진입. 로그인/계정 생성 선택.
- **요소**: 로고, 서비스명 "치즈모아", 태그라인 "사진 정리부터 공유까지 한 번에", `[로그인]` 버튼, `[계정 생성]` 버튼.
- **액션**: 로그인→01-1 / 계정 생성→01-2.
- **상태**: 이미 로그인 토큰 보유 시 자동으로 홈(02)으로.

### 01-1. 로그인 · node `243:33`
- **목적**: 기존 제작자 로그인.
- **요소**: 닉네임 입력, PIN(숫자 4자) 입력, `[로그인]` 버튼.
- **액션**: 제출 → `POST /auth/login` → 성공 시 홈(02).
- **상태**: 빈 값/형식 오류(PIN≠4자리) 비활성, 인증 실패 에러 메시지.
- **API**: `POST /auth/login`

### 01-2. 계정 생성 · node `243:2`
- **목적**: 신규 제작자 가입.
- **요소**: 닉네임 입력("닉네임 입력"), PIN(숫자 4자, "PIN 번호 입력"), `[생성]` 버튼.
- **액션**: 제출 → `POST /auth/signup` → 성공 시 홈(02).
- **상태**: 닉네임 중복/PIN 형식 검증, 에러 메시지.
- **API**: `POST /auth/signup`

### 02. 홈 / 내 모임 · node `211:1357`(목록) · `211:1396`(빈 상태)
- **목적**: 참여 중인 모임 목록.
- **요소**: 헤더 "내 모임 / 참여 중인 모임을 확인하세요", 모임 카드 리스트(모임명, `인원 N명 · 이벤트 M개`), `[＋ 모임 만들기]`, `[모임 참여하기]`.
- **액션**: 카드 탭→모임 상세(05) / 모임 만들기→03 / 모임 참여하기→02-1(모달).
- **상태**: **빈 상태**(`211:1396`) = 모임 카드 없이 두 버튼만.
- **확정(CHMO-111)**: 와이어프레임의 카드 내 `모임 설정 ⚙`는 표시하지 않는다 — 모임 설정(이름 수정)은 모임 상세(05)의 `⚙`로 일원화. 카드 전체가 상세 이동 탭 영역이라 내부 `⚙`는 오탭을 유발하고, 디자인 시안(dc.html §05 GroupCard)도 이 형태.
- **확인 필요(플래그)**: 와이어프레임의 `관리자` 배지·`📌` 고정 아이콘은 권한 등급 제거 결정에 따라 MVP에서 표시하지 않음(추후 재도입 여지).
- **API**: `GET /groups`

### 02-1. 모임 참여(모달) · node `211:1520` / `211:1532`
- **목적**: 초대받은 모임에 **제작자로 합류**(선생님 초대 수락).
- **요소**: "모임 참여 / 초대받은 모임의 비밀번호를 입력하세요", 비밀번호 입력, `[참여]`.
- **진입 경로**: 홈의 `모임 참여하기` 또는 초대 링크(`/join/:groupKey`)로 진입 → 해당 모임 식별 + 비밀번호 입력.
- **액션**: 제출 → `POST /groups/join` → 성공 시 모임 상세(05).
- **상태**: 비밀번호 불일치 에러.
- **API**: `POST /groups/join`

### 03. 모임 만들기 · node `211:1411`
- **목적**: 새 모임 생성.
- **요소**: 헤더 `‹ 홈`, "모임 만들기", 배지 "무료 · 용량 차면 업그레이드"(표시만), 모임 이름(예: "햇살반 학부모"), 비밀번호("참여 시 입력할 비밀번호"), 안내 카피, `[모임 만들기]`.
- **액션**: 제출 → `POST /groups` → 모임 상세(05).
- **상태**: 이름/비번 필수.
- **MVP 메모**: 업그레이드/결제는 미구현(카피만 표시).
- **API**: `POST /groups`

### 05. 모임 상세 = 이벤트 목록 · node `211:1443`(목록) · `211:1432`/`211:1505`(빈/신규)
- **목적**: 모임 안 이벤트 목록 + 모임 액션 허브.
- **요소**: 모임명("햇살반"), 이벤트 카드(이벤트명, `날짜 · 사진 N장 · 인원`, **상태 배지**, 설정 `⚙`), `[+ 선생님 초대]`, `[+ 이벤트 생성]`, `[⧉ 학부모님에게 공유]`.
- **이벤트 상태 배지**: `분류중`, `공개 준비`(검토완료), `공개 완료`, `NEW`(사진 0장 신규). (진행률 % 미표시 — MVP 제외)
- **액션**: 이벤트 탭→(사진 있으면)08 /(없으면)06-E · 선생님 초대→초대 화면 · 이벤트 생성→06-M(모달) · 모임 설정 `⚙`→**모임 이름 수정**(`PATCH /groups/:id`, 이름만) · `⧉ 학부모님에게 공유`→**모임 학부모 공유**(링크+학부모 전용 비밀번호, `GET /groups/:id/share`).
- **학부모 공유(확정)**: **모임 단위** 링크 + 학부모 전용 비밀번호(제작자 합류용 모임 비밀번호와 별개, 모임 생성 시 자동). 학부모는 이 링크로 들어와 **공개된 이벤트만** 골라 본다(→ 15-L).
- **확정(CHMO-112)**: ① 초대·학부모 공유는 별도 페이지가 아니라 **05 위 바텀시트**(`/groups/:id/invite` 라우트 제거). ② 이벤트 카드 메타는 `날짜 · 사진 N장`만 — 인원(원아 수)은 이벤트 목록 API에 없어 미표시. ③ 이벤트 카드 내 `⚙`는 대응 액션이 없어 미표시(이벤트 이름 수정은 08의 `✎`). ④ 와이어프레임 `211:1432`의 `모임삭제`는 스펙·API에 없어 MVP 미표시. ⑤ 이벤트 생성 모달(06-M)은 CHMO-113 — 그전까지 `[+ 이벤트 생성]`은 토스트 안내.
- **API**: `GET /groups/:id`, `PATCH /groups/:id`(이름 수정), `GET /groups/:id/events`, `GET /groups/:id/share`(학부모 공유)

### 초대(선생님 초대하기) · node `211:1556`
- **목적**: **제작자(선생님) 모임 합류용** 초대 정보 공유.
- **요소**: "초대하기 / 참여자를 초대하세요", 모임 비밀번호(예: `482AVX`), 참여 링크(예: `.../join/HAETSAL`), `⧉`(복사), `[링크복사]`, `[카카오톡 공유]`.
- **액션**: 링크/비밀번호 복사·공유. 받은 사람은 02-1에서 비밀번호 입력해 합류.
- **메모**: 이 화면은 **학부모 공개와 무관**(학부모 공유는 **모임 단위 별도** → 05의 `⧉ 학부모님에게 공유` / `GET /groups/:id/share`). 도메인 `classpic.app`은 치즈모아 도메인으로 교체(placeholder).
- **확정(CHMO-112)**: 와이어프레임(공유 시트) 그대로 **05 위 바텀시트**로 표시 — 별도 페이지·라우트 없음. 카카오톡 공유는 목(토스트). 학부모 공유 시트도 같은 폼(F6.3 — 링크복사·카카오톡 공유 포함).
- **API**: `GET /groups/:id/invite`

### 06-M. 이벤트 이름 입력(모달) · node `211:1544`
- **목적**: 새 이벤트 생성.
- **요소**: "이벤트 이름", 안내 "오늘 날짜가 들어가 있어요...", 이름 입력(기본값 = 오늘 날짜, 예 `2026-06-27`), `[생성]`.
- **액션**: 생성 → `POST /groups/:id/events` → 빈 이벤트(06-E).
- **API**: `POST /groups/:id/events`

### 06-E. 빈 이벤트 · node `211:1572`
- **목적**: 사진 없는 이벤트의 초기 화면.
- **요소**: 헤더 `‹ 모임명`, 이벤트명, 📷 일러스트, "아직 사진이 없어요", `[사진 업로드]`.
- **액션**: 사진 업로드→06-U.
- **API**: `GET /events/:id`

### 06-U. 사진 업로드 · node `211:1584`
- **목적**: 다중 사진 업로드 + 분석 옵션 설정 후 사진 분류 시작.
- **요소**: 헤더 `‹ 이벤트명`, "사진 업로드", `선택됨 N장`, 썸네일 그리드(선택 체크 ✓), 토글 **눈감은 사진 제외(제외 ON)**, 토글 **흔들린 사진 제외(제외 ON)**, 안내 "제외한 사진은 분류가 어려워요 사진첩으로 이동해요", `[사진 분류하기]`.
- **액션**: 사진 선택 → 업로드(**presign `POST /events/:id/photos/presign` → S3 직접 PUT**, 등록은 S3 이벤트로 자동) → `[사진 분류하기]` `POST /events/:id/analyze`(제외 옵션 함께 전송) → 배지 `분류중` → 완료(분석중 화면 5초 자동 폴링 — CHMO-244) 시 이벤트 상세(08).
- **상태**: 선택 0장 시 분석 비활성, 업로드 진행률(S3 직접 PUT). (사진 분류 진행률 % 없음 — MVP 제외. 분석중 화면은 5초 간격 자동 폴링으로 완료 시 08로 자동 전환 — CHMO-244)
- **확인 필요(플래그)**: 업로드 카피는 "제외→분류가 어려워요"라고 하지만, 08 화면은 `눈감은 사진`/`흔들린 사진` 앨범이 별도로 존재. → 기능 명세에서 "품질 제외 사진은 각 품질 앨범으로 라우팅"으로 정의(아래 08 참고).
- **API**: `POST /events/:id/photos/presign`(→ S3 직접 PUT), `POST /events/:id/analyze`, `GET /events/:id/analysis`

### 08. 이벤트 상세 = 앨범 그리드 · node `211:1619`
- **목적**: AI 분류 결과 확인·검수 허브.
- **요소**: 헤더 `‹ 이벤트 목록`, "이벤트 상세", 이벤트명("6.15 운동회 오전"), `✎ 수정`, **앨범 카드 그리드(2단)**:

- **확정(CHMO-118)**: ① 헤더는 기존 서브형 관례(`‹ 홈` + 타이틀 "설정")로 적용, "프로필 편집"은 본문 제목 — 와이어프레임의 `‹ 설정`은 별도 설정 목록 화면 전제인데 MVP IA에는 그 화면이 없음. ② 닉네임은 `GET /me`로 프리필. PIN은 응답에 없어 빈 칸 시작 — **입력했을 때만** `PATCH /me`에 포함(빈 값 = 닉네임만 변경). `[저장]`은 **실제 변경이 있을 때만 활성**(닉네임 수정 또는 새 PIN 4자리 완성 — 무변경·PIN 미완성이면 비활성). 저장 중엔 입력 필드도 잠근다(늦은 응답이 입력을 덮지 않게). ③ **로그아웃 버튼 포함**(하단 `[저장]` 위 secondary, MVP 스코프 확정 사항) — 앱 유일의 로그아웃 표면이라 프로필 로딩/실패 중에도 상시 노출. 토큰 삭제 후 01 로그인 진입으로 복귀.
- **API**: `GET /me`, `PATCH /me`

## 5. 구현 시 확정 필요 항목(요약)
1. 홈의 `관리자` 배지/`📌` 고정 — 권한 제거에 따라 미표시(추후 재도입 여지).
2. `⧉ 학부모님에게 공유` — **모임 단위 공유로 확정**(05에 위치, `GET /groups/:id/share`). 학부모 전용 비밀번호(모임 비밀번호와 별개, 생성 시 자동), 비번 후 공개 이벤트 목록(15-L)에서 선택. (더 이상 미정 아님 · 신규 화면 15-L은 와이어프레임 없음 → 시안 필요)
3. 품질 제외 사진 라우팅 — `눈감은 사진`/`흔들린 사진` 앨범으로(08 기준), 업로드 카피와 정합 맞추기.
4. `공통(common)` 앨범 정의 — 앨범↔사진 다대다 확정으로 "다수 인물"은 각 아이 앨범에 모두 들어가므로, `공통`은 **특정 아이로 분류되지 않은 사진(단체/배경 등)**으로 좁혀 정의(구현 시 확정).
5. 도메인 `classpic.app` → 치즈모아 도메인(placeholder).

> **참고**: 다대다(사진=여러 앨범), 삭제=현재 앨범에서만 제거(연결 해제), 옮기기=연결 교체, 인물 이름=모임 단위 공유(rename 그룹 전파)는 ERD 반영으로 **확정**됨(더 이상 미정 아님).
