# 치즈모아 MVP — 기능 명세서 (Feature Spec)

> 함께 보기: [화면 명세](./screen-spec.md) · [API 명세서](./api-spec.md) · [서비스 소개](./mvp-explanation.md)
> 범위: 프론트엔드(React 모바일웹) 개발 착수용. BE/AI는 본 문서 범위 밖이며 API는 [계약](./api-spec.md)으로만 정의.

---

## 1. 제품 한 줄 / 핵심 가치

선생님이 행사 사진을 올리면 **AI가 아이별 앨범으로 자동 분류**하고, 선생님은 **빠르게 검수**한 뒤 **이벤트 단위로 공개**한다. 학부모는 **링크 + 비밀번호**로 들어와 자기 아이 앨범을 **조회·다운로드**한다. (소개 문서 기반)

핵심: **"선생님은 올리기만, 정리는 자동, 학부모는 받아보기만."**

---

## 2. 사용자 유형

| 유형 | 인증 | 할 수 있는 것 |
|---|---|---|
| **제작자(선생님)** | 닉네임 + 4자리 PIN 로그인 | 모임 생성/참여, 이벤트 생성, 업로드, AI 분석, 검수, 공개. **권한 등급 없음 — 모임 멤버는 전원 동일.** |
| **학부모(뷰어)** | 무로그인. **모임** 공유 링크 + 학부모 전용 비밀번호 | 비번 입력 → 공개된 이벤트 목록에서 선택 → 앨범 조회, 사진 다운로드/공유. |

---

## 3. 도메인 모델 / 용어

```
User(제작자) >──< Membership >──< Group(모임) ──< Event(이벤트) ──< Album(앨범) >──< Photo(사진)
                                  └──< RepresentativeVector(대표 벡터 · 모임 단위 인물)
Album.personId ─▶ RepresentativeVector     (인물 앨범은 모임 단위 인물을 참조; 표시 이름도 여기서 옴)
```

> ERD 기준 관계: **유저↔모임 = N:M(멤버십)**, 모임─이벤트─앨범 = 1:N 체인, **앨범↔사진 = N:M**(한 사진이 여러 앨범에 속함), 대표 벡터 = 모임에 속하고 인물 앨범이 참조.

| 용어 | 정의 |
|---|---|
| **모임(Group)** | 제작자들이 모인 단위(예: 햇살반). 이름 + **모임 비밀번호**(제작자 합류용) + **학부모 공유 링크·학부모 전용 비밀번호**(모임 단위, 생성 시 자동) 보유. 여러 이벤트·여러 멤버. |
| **멤버십(Membership)** | 제작자↔모임 **N:M** 관계(한 모임에 여러 제작자, 한 제작자가 여러 모임). 등급/권한 구분 없음. BE 내부, FE엔 `Group.memberCount`로만. |
| **이벤트(Event)** | 한 행사의 사진 묶음(예: 6.15 운동회 오전). **공개(publish) 단위** — `published`가 되면 모임 학부모 공유 목록에 노출. 상태 보유(공유 링크/비밀번호는 모임 단위로 이동). |
| **대표 벡터(RepresentativeVector)** | **모임 단위 아이 정체성** — 얼굴 임베딩 + **이름** 보유. 이벤트가 달라도 같은 아이를 일관 인식. **BE 내부(FE 비노출)**, 인물 앨범이 `personId`로 참조하고 이름을 가져옴. |
| **앨범(Album)** | 이벤트 안에서 사진이 분류된 묶음. 종류는 아래 표. 검토 상태는 **사진 단위**로 저장(앨범 카드 표시는 미검토 사진 유무로 파생). 인물 앨범은 `personId`(모임 단위 인물)를 가짐. |
| **사진(Photo)** | 업로드된 이미지. 품질 플래그(눈감음/흔들림) 보유. **앨범과 다대다** — 여러 아이가 같이 찍히면 각 아이 앨범에 모두 속함. |

### 앨범 종류

| type | 라벨 | 설명 | 뷰어 노출 |
|---|---|---|---|
| `person` | 아이 이름 | AI가 같은 인물로 군집화. `personId`(모임 단위 인물) 보유, **이름은 모임 단위 공유**(이벤트 간 동일) | ✅ |
| `common` | 공통 | 다수 인물 또는 특정 인물로 특정되지 않은 사진(*가정 — 구현 시 확정*) | ✅ |
| `uncertain` | 분류가 어려워요 | AI가 인물 판단을 못 한 사진 | ❌(내부 검수용) |
| `eyes_closed` | 눈감은 사진 | 눈감음으로 품질 제외된 사진 | ❌ |
| `blurry` | 흔들린 사진 | 흔들림으로 품질 제외된 사진 | ❌ |

> 업로드 시 "눈감은 사진 제외"/"흔들린 사진 제외"가 ON이면 해당 사진은 인물 앨범이 아니라 `eyes_closed`/`blurry` 앨범으로 라우팅된다. (와이어프레임 06-U의 "분류가 어려워요로 이동" 카피는 08 화면의 분리 앨범 기준으로 정정.)

## 4. 상태(State machine)

### 이벤트 상태
```
empty(빈, 사진0) → analyzing(분석중) → review(검수중) → ready(공개 준비 = 검토완료) → published(공개됨)
```
- `empty`: 생성 직후, 사진 0장. 배지 `NEW`.
- `analyzing`: 업로드+AI 분석 진행 중. 배지 `분석중`. (진행률 % 미표시 — MVP 제외)
- `review`: 분석 완료, 검수 진행 가능(미검토 사진 존재).
- `ready`: 모든 사진 검토완료. 배지 `공개 준비`.
- `published`: 공개 실행 완료. 배지 `공개 완료`. 학부모 링크 활성.
- (사진 추가 시 재분석: `review`/`ready`는 `analyzing`으로 회귀. **`published`는 공개를 유지한 채 증분 분석**(상태 전이 없음) — 공개 후에도 사진 추가·재분석·이동 가능하며, 새 사진은 검토 완료 전까지 뷰어에 비노출.)
- 재분석은 **증분**: 아직 앨범에 속하지 않은 사진만 분류하고, 기존 앨범·수동 배치·검토 상태는 보존한다.

### 사진 검토 상태
```
unreviewed(미검토) → reviewed(검토완료)
```
- **검토는 사진 단위.** 앨범의 `검토 완료`는 앨범 내 전 사진 일괄 처리 액션(앨범 자체는 검토 상태를 갖지 않음).
- **미검토 사진은 학부모 뷰어에 비노출**(서버가 필터링해 응답).
- UI(앨범 카드): 갈색 실선 = 앨범 내 미검토 사진 0장 · 회색 점선 = 미검토 사진 존재.

## 5. 기능 명세 (유저스토리 + 수용 기준)

> 형식: **US** 유저스토리 / **AC** 수용 기준 / **화면** / **API**

### F1. 인증

**F1.1 계정 생성**
- US: 제작자로서 닉네임과 4자리 PIN으로 계정을 만든다.
- AC: PIN은 숫자 4자리만 허용 · 닉네임 중복 시 에러 · 성공 시 자동 로그인되어 홈 진입.
- 화면: 01-2 · API: `POST /auth/signup`

**F1.2 로그인**
- US: 닉네임 + PIN으로 로그인한다.
- AC: 자격 불일치 시 에러 · 성공 시 토큰 저장 후 홈 · 토큰 보유 시 01 진입을 건너뛰고 홈.
- 화면: 01, 01-1 · API: `POST /auth/login`

**F1.3 프로필 편집**
- US: 닉네임과 PIN을 변경한다.
- AC: PIN 4자리 검증 · 저장 시 즉시 반영.
- 화면: 설정/프로필 편집 · API: `GET /me`, `PATCH /me`

### F2. 모임

**F2.1 모임 목록(홈)**
- US: 참여 중인 모임을 한눈에 본다.
- AC: 각 카드에 모임명·`인원 N명·이벤트 M개` 표시 · 모임 0개면 빈 상태(만들기/참여 버튼만).
- 화면: 02 · API: `GET /groups`

**F2.2 모임 만들기**
- US: 이름과 비밀번호로 새 모임을 만든다.
- AC: 이름·비밀번호 필수 · 생성자는 자동으로 멤버 · 생성 후 모임 상세로 이동.
- 화면: 03 · API: `POST /groups`

**F2.3 선생님 초대 / 모임 참여(제작자 합류)**
- US: 다른 선생님을 모임에 합류시킨다 / 초대받아 합류한다.
- AC: 초대 화면에서 **모임 비밀번호 + 참여 링크 + 카카오톡 공유** 제공 · 합류자는 링크로 진입 후 **모임 비밀번호** 입력으로 합류 · 합류자는 기존 멤버와 동일 능력(업로드·검수·공개) · **학부모 공개와 별개 입구.**
- 화면: 초대(`211:1556`), 02-1 · API: `GET /groups/:id/invite`, `POST /groups/join`

**F2.4 모임 이름 수정**
- US: 모임 이름을 바꾼다.
- AC: 모임 설정(`⚙`)에서 이름 변경 · 저장 즉시 반영 · **이름만 수정 가능**(비밀번호·멤버 등 불가) · 빈 이름 불가.
- 화면: 05(모임 설정 `⚙`, 세부는 화면 제작 시 확정) · API: `PATCH /groups/:id`

### F3. 이벤트

**F3.1 이벤트 생성**
- US: 모임 안에 새 이벤트를 만든다.
- AC: 이름 기본값 = **오늘 날짜** · 수정 후 생성 · 생성 직후 빈 이벤트(사진 0장, 배지 `NEW`).
- 화면: 06-M → 06-E · API: `POST /groups/:id/events`

**F3.2 이벤트 목록/상태 표시**
- US: 모임의 이벤트들을 상태와 함께 본다.
- AC: 카드에 이벤트명·날짜·사진 수 · 상태 배지(`분석중`/`공개 준비`/`공개 완료`/`NEW`).
- 화면: 05 · API: `GET /groups/:id/events`

**F3.3 이벤트 이름 수정**
- US: 이벤트 이름을 바꾼다.
- AC: 상세에서 `✎ 수정` · 저장 즉시 반영.
- 화면: 08 · API: `PATCH /events/:id`

### F4. 업로드 / AI 분석

**F4.1 사진 업로드(presigned 직행)**
- US: 이벤트에 사진을 여러 장 올린다.
- AC: 다중 선택 · 선택 수 표시(`선택됨 N장`) · 0장이면 분석 비활성 · **presign으로 업로드 URL 받고 → S3에 직접 PUT**(파일은 서버/Lambda 안 거침) · 업로드 진행률 표시(FE가 S3 PUT 진행률 측정) · 등록은 S3 이벤트로 자동(별도 완료 호출 없음).
- 화면: 06-U · API: `POST /events/:id/photos/presign`, (S3 직접 PUT)

**F4.2 분석 옵션**
- US: 눈감은/흔들린 사진을 자동 제외한다.
- AC: 두 토글(기본 ON) · ON 시 해당 사진은 인물 앨범 대신 `eyes_closed`/`blurry`로 라우팅.
- 화면: 06-U · API: **분석 요청 파라미터**(`excludeEyesClosed`, `excludeBlurry` in `POST /events/:id/analyze`)

**F4.3 AI 분석 실행**
- US: 올린 사진을 아이별로 자동 분류한다.
- AC: `[AI 분석]` 실행 → 이벤트 `analyzing`(배지 `분석중`) → 완료 시 `review`로 전환, 이벤트 상세(08) 앨범 그리드 생성. **진행률 %·자동 폴링 없음**(MVP 제외) — 완료는 화면 재진입/새로고침 시 상태로 확인.
- 화면: 06-U → 08 · API: `POST /events/:id/analyze`, `GET /events/:id/analysis`

### F5. 앨범 / 검수

**F5.1 앨범 그리드 조회**
- US: 분류 결과를 앨범 단위로 확인한다.
- AC: **2단 구성** — ① **앨범 그리드**: 인물 앨범(이름+장수+검토상태)과 `분류가 어려워요`(`uncertain`)를 **동일한 카드 모양(커버 썸네일 포함)으로 같은 그리드**에 배치(분류가 어려워요는 선생님이 아이 앨범으로 재분류할 대상) · ② **품질 제외**(`눈감은 사진`/`흔들린 사진`)만 하단 별도 섹션 · 테두리 색으로 검토 상태 구분.
- 화면: 08 · API: `GET /events/:id/albums`

> 사진은 **여러 앨범에 속할 수 있음**(다대다). 아래 삭제/이동 의미는 이를 전제로 한다.

**F5.2 사진 선택·삭제(앨범에서 빼기)**
- US: 앨범에서 잘못 분류된/불필요한 사진을 그 앨범에서 뺀다.
- AC: 선택 모드 · 다중 선택 · 삭제 확인(warn) · 삭제 = **현재 앨범에서만 제거(연결 해제)** · **다른 앨범엔 그대로 유지**(다대다) · 그 앨범에만 있던 사진이면 이후 어떤 앨범에도 안 남음(마지막 연결 해제 = 완전 삭제, 휴지통 없음) · 즉시 반영.
- 화면: 09 · API: `DELETE /photos`

**F5.3 사진 이동(분류 보정)**
- US: 다른 아이 앨범으로 사진을 옮긴다.
- AC: `옮기기` → 바텀시트 · **유사도 높은 순 추천** 앨범 목록(%) + `공통 사진첩` · 대상 선택 시 **현재 앨범에서 빼고 대상 앨범으로 이동**(연결 교체, 복사 아님) · 반영.
- 화면: 09 → 09-1 · API: `GET /albums/:id/move-suggestions`, `POST /photos/move`

**F5.4 검토 완료 처리**
- US: 확인이 끝난 앨범을 검토완료로 표시한다.
- AC: 앨범의 `검토 완료` = **앨범 내 전 사진 일괄 reviewed 처리**(검토 상태는 사진 단위 저장) · 상태/테두리 즉시 반영 · 전 사진 완료 시 이벤트 `ready`.
- 화면: 08, 09 · API: `PATCH /albums/:id`

**F5.5 사진 추가**
- US: 검수 중 사진을 더 올린다.
- AC: `+ 사진 추가` → 업로드(06-U) 재사용 · 추가분 분석 후 앨범 반영.
- 화면: 08 → 06-U

**F5.6 인물 이름 변경(rename)**
- US: 인물 앨범의 아이 이름을 바꾼다(예: 클러스터 라벨 → 실제 이름).
- AC: 인물 앨범 이름 변경 시 그 이름은 **모임 단위 인물(`personId`)에 저장** → **같은 모임의 모든 이벤트에 있는 같은 아이 앨범 이름이 함께 바뀐다** · 특수 앨범은 이름 변경 불가 · 변경 즉시 반영(같은 `personId`를 쓰는 다른 이벤트 앨범도 갱신).
- 화면: 08(인물 앨범) · API: `PATCH /albums/:id` (`name`)

### F6. 공개

**F6.1 공개 전 검수(요약)**
- US: 공개 직전 전체를 요약으로 확인한다.
- AC: 통계(사진 수/앨범 수/검토완료 **사진** n/n/분류 애매 수) + 미리보기(학부모 뷰 프리뷰).
- 화면: 14 · API: `GET /events/:id/review-summary`

**F6.2 이벤트 공개하기**
- US: 이벤트를 학부모에게 공개한다.
- AC: `[공개하기]` → 이벤트 `published` → **모임 학부모 공유 목록에 노출**(이벤트별 공유 링크는 없음, 공유는 모임 단위) · 미검토 사진이 있어도 공개 가능하되 경고 노출(미검토 사진은 뷰어에 비노출, *경고 정책 확정: `?force=true` 없이는 `409 HAS_UNREVIEWED_PHOTOS` 반환 → FE가 확인 후 force 재요청*) · **공개 후에도 사진 추가·재분석·이동 가능**(상태는 `published` 유지, 새 사진은 검토 후 노출).
- 화면: 14 · API: `POST /events/:id/publish`

**F6.3 학부모에게 공유(모임 단위)**
- US: 모임의 학부모 공유 링크/비밀번호를 학부모에게 보낸다.
- AC: **모임 단위** 공유 링크 + **학부모 전용 비밀번호**(제작자 합류용 모임 비밀번호와 별개, 모임 생성 시 자동 발급) · 링크복사/카카오톡 공유 · 학부모는 이 링크로 들어와 **공개된 이벤트만** 골라 본다.
- 화면: 05(학부모 공유 액션) · API: `GET /groups/:id/share`

### F7. 학부모 뷰어(무로그인)

**F7.1 잠금 해제**
- US: 받은 **모임** 공유 링크의 비밀번호를 넣어 본다.
- AC: 링크(`/share/:token`, 모임 공유) → 학부모 전용 비밀번호 입력 → 일치 시 뷰어 세션 발급(모임 범위) · 불일치 에러.
- API: `POST /share/:token/unlock`

**F7.2 공개 이벤트 목록 조회·선택**
- US: 모임의 공개된 이벤트 목록을 보고 하나를 고른다.
- AC: **공개(`published`)된 이벤트만** 목록 노출(이벤트명·날짜·사진 수·커버) · 공개 이벤트 없으면 빈 상태 · 이벤트 선택 시 해당 이벤트 앨범(15)으로.
- 화면: 15-L · API: `GET /share/:token`

**F7.3 공개 이벤트 앨범 조회**
- US: 선택한 이벤트의 앨범을 본다.
- AC: **인물 + 공통 앨범만** 노출(특수 앨범 비노출) · `공개됨` 배지.
- 화면: 15 · API: `GET /share/:token/events/:eventId`

**F7.4 인물 앨범 조회·다운로드·공유**
- US: 우리 아이 앨범을 보고 받는다.
- AC: 앨범 사진 그리드 · 다운로드(일괄/개별) · 공유(링크 재공유) · **다운로드 한도 없음**.
- 화면: 16 · API: `GET /share/:token/events/:eventId/albums/:albumId`, 다운로드 엔드포인트

## 6. MVP 제외 (소개 문서 기준)
